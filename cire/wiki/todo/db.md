# cire/db

Schema and migration work. See [[monorepo-structure]] for how this package fits into the dependency graph.

- [x] **`events.event_image_key` column** (migration `0019_event_image.sql`, `feat/cire-event-images`) ‚Äî ONE optional image per event. A single nullable `text` column added in place via `ALTER TABLE events ADD event_image_key text`; stores the R2 object **key** (not a URL), exactly like `wedding_invite_customisations.hero_image_key`. NULL ‚áí the event renders text-only at every breakpoint (no empty image half). Additive + self-backfilling (every existing row defaults NULL ‚Äî live events have no image until one is uploaded through the organiser portal). Events have NO `updated_at`, so the served image's cache version is derived **server-side** from the key (FNV-1a digest in `versionFromKey`): a re-upload mints a fresh uuid-suffixed key ‚áí a fresh version ‚áí never served stale, and the client `?v=` is never trusted for keying (S-M1). Verified: all 19 migrations apply to a fresh local D1 and the table shape matches `schema.ts`; LOCKSTEP mirror updated in `cire/api/src/db/setup.ts`. See `[[api]]` + `[[web]]` + `[[invite-builder]]`.
